       def _start_services():
           """Start Docker Compose services and wait for health."""
           click.echo("  Starting Docker Compose services...")
           project_root = _get_project_root()
           compose_file = project_root / "docker" / "docker-compose.yml"
           if not compose_file.exists():
               click.secho(f"  docker-compose.yml not found at {compose_file}", fg="red")
               return
           try:
               result = subprocess.run(
                   ["docker", "compose", "-f", str(compose_file),
                    "--project-directory", str(project_root / "docker"),
                    "up", "-d"],
                   capture_output=True, text=True, timeout=120
               )
       ```

       <param name="proxy" value="$${HOLLER_TRUNK_HOST}"/>
       ```

    2. **Update `docker/docker-compose.yml`** — Add environment variable injection into the FreeSWITCH service so it receives the trunk credentials from the host. Add an `environment:` block to the freeswitch service:
       ```yaml
       services:
         freeswitch:
           build:
             context: ./freeswitch
             args:
               SIGNALWIRE_TOKEN: ${SIGNALWIRE_TOKEN}
           network_mode: "host"
           environment:
             - HOLLER_TRUNK_HOST=${HOLLER_TRUNK_HOST:-}
             - HOLLER_TRUNK_USER=${HOLLER_TRUNK_USER:-}
             - HOLLER_TRUNK_PASS=${HOLLER_TRUNK_PASS:-}
           volumes:
             - ../config/freeswitch:/etc/freeswitch
             - freeswitch-logs:/var/log/freeswitch
           restart: unless-stopped
       ```
       Use `${VAR:-}` syntax so compose doesn't error when vars are unset — they'll be empty strings, which is fine (FreeSWITCH won't register the gateway if creds are blank).

       FreeSWITCH reads environment variables via `$${VAR_NAME}` syntax in its XML config. When the container has `HOLLER_TRUNK_HOST` in its environment, `$${HOLLER_TRUNK_HOST}` in external.xml resolves to that value.

    3. **Update `config/freeswitch/vars.xml`** — Add default empty values for the trunk variables so FreeSWITCH does not fail on startup when trunk is not yet configured:
       Add before the closing `</include>`:
       ```xml
       <X-PRE-PROCESS cmd="set" data="HOLLER_TRUNK_HOST="/>
       <X-PRE-PROCESS cmd="set" data="HOLLER_TRUNK_USER="/>
       <X-PRE-PROCESS cmd="set" data="HOLLER_TRUNK_PASS="/>
       ```
       These defaults are overridden when environment variables are present (FreeSWITCH env vars override X-PRE-PROCESS set).
