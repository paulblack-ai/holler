    Create `holler/core/compliance/consent_db.py` per D-14, D-16.

    Class: `ConsentDB`
    - `__init__(self, db_path: str)` — store path, `self._db = None`
    - `async def initialize(self) -> None` — open persistent aiosqlite connection, enable WAL mode, create `consent` table if not exists. Schema per Research SQLite Schema section:
      ```sql
      CREATE TABLE IF NOT EXISTS consent (
          id          INTEGER PRIMARY KEY AUTOINCREMENT,
          phone_number TEXT NOT NULL,
          consent_type TEXT NOT NULL CHECK(consent_type IN ('express', 'written')),
          granted_at   TEXT NOT NULL,
          revoked_at   TEXT,
          source       TEXT NOT NULL CHECK(source IN ('api', 'call', 'sms', 'dtmf')),
          call_uuid    TEXT,
          created_at   TEXT NOT NULL DEFAULT (datetime('now', 'utc'))
      );
      ```
      Note: Add 'dtmf' to the source CHECK constraint (D-15 specifies DTMF as an opt-out channel).
    - `async def record_consent(self, phone_number: str, consent_type: str, source: str = "api", call_uuid: Optional[str] = None) -> None` — INSERT new row with `granted_at=datetime.now(timezone.utc).isoformat()`, `revoked_at=NULL`.
    - `async def record_optout(self, phone_number: str, source: str = "dtmf", call_uuid: Optional[str] = None) -> None` — INSERT new row with `consent_type="express"`, `granted_at` of original consent (or now), `revoked_at=datetime.now(timezone.utc).isoformat()`.
    - `async def has_consent(self, phone_number: str) -> bool` — Query: find the most recent row for this phone_number (ORDER BY id DESC LIMIT 1). Return True only if `revoked_at IS NULL`.
    - `async def close(self) -> None` — close the aiosqlite connection.

    Key design: Append-only. NEVER use UPDATE or DELETE on consent rows. Opt-outs are new INSERT rows with `revoked_at` populated. This is legally required per D-14.

    Use `from __future__ import annotations` and `from typing import Optional`. Use `datetime.now(timezone.utc).isoformat()` for timestamps.

    Tests: Use in-memory SQLite (`:memory:`) or temp files (via `tmp_path` fixture). Each test initializes a fresh ConsentDB, runs operations, asserts state.

    Create `holler/core/compliance/audit.py` per D-20, D-21, D-22 and Research Pattern 8.

    Class: `AuditLog`
    - `__init__(self, log_dir: str, db_path: str)` — store paths, `self._db = None`
    - `async def initialize(self) -> None` — open persistent aiosqlite connection, WAL mode, create audit_log table:
      ```sql
      CREATE TABLE IF NOT EXISTS audit_log (
          id          INTEGER PRIMARY KEY AUTOINCREMENT,
          logged_at   TEXT NOT NULL,
          call_uuid   TEXT,
          session_uuid TEXT,
          check_type  TEXT NOT NULL,
          destination TEXT,
          result      TEXT NOT NULL CHECK(result IN ('allow', 'deny')),
          reason      TEXT,
          did         TEXT,
          log_file    TEXT
      );
      CREATE INDEX IF NOT EXISTS idx_audit_call_uuid ON audit_log(call_uuid);
      CREATE INDEX IF NOT EXISTS idx_audit_destination ON audit_log(destination);
      CREATE INDEX IF NOT EXISTS idx_audit_logged_at ON audit_log(logged_at);
      ```
    - `def _today_log_path(self) -> Path` — return `{log_dir}/compliance-{YYYY-MM-DD}.jsonl`
    - `async def write(self, entry: dict) -> None` — Per D-21:
      1. Add `logged_at = datetime.now(timezone.utc).isoformat()` to entry
      2. Append one JSON line to today's JSONL file (create dir if needed, open in append mode)
      3. Insert row into SQLite audit_log table
      The JSONL write is the primary immutable record. SQLite is the derived index.
    - `async def query_by_call_uuid(self, call_uuid: str) -> list` — SELECT * FROM audit_log WHERE call_uuid = ? ORDER BY logged_at.
    - `async def close(self) -> None` — close connection.

    JSONL files must be write-once (no updates, no deletes) per D-22. The `open(path, "a")` mode ensures append-only semantics at the OS level.

    Use `json.dumps(entry)` for each line. Use `pathlib.Path` for directory creation with `mkdir(parents=True, exist_ok=True)`.

    Tests: Use `tmp_path` for both log directory and SQLite database. Verify JSONL content is parseable, file naming follows date pattern, SQLite index is queryable.
